# ข้อกำหนดโปรแกรมรายบท (Chapter Program Specification)

เอกสารนี้ระบุรายละเอียดเวิร์กโฟลว์หลักและกระบวนการทำงานย่อย (Subroutines) ของระบบ `ServiceTransfer` ซึ่งเดิมพัฒนาด้วย Visual Basic 6 เนื่องจากระบบนี้เป็นบริการเบื้องหลัง (Background Service) ไม่ใช่ API จึงจัดรูปแบบฟังก์ชันในลักษณะโมดูลและลูปแทน

## ฟังก์ชัน / กระบวนการหลัก (Core Functions)

### 1. `otmForm_Timer` (ลูปหลักสำหรับการซิงค์ข้อมูล)
- **วิธีการ (Method/Type):** Internal Timer Event (ทำงานเป็นลูปเบื้องหลัง)
- **ข้อมูลที่ส่งเข้า (Request Payload):** ไม่มี (ถูกเรียกอัตโนมัติตามช่วงเวลา)
- **ข้อมูลส่งกลับ (Response Payload):** ไม่มี
- **สรุปตรรกะการทำงาน (Core Logic):**
  1. ตัวจับเวลาของแอปพลิเคชันจะเรียกฟังก์ชันนี้ขึ้นมาเป็นระยะ
  2. เรียกใช้ `SP_CHKbWebservice` เพื่อตรวจสอบว่าฐานข้อมูลเซิร์ฟเวอร์ส่วนกลางสามารถเชื่อมต่อได้หรือไม่ (Online)
  3. หากเชื่อมต่อได้ จะสืบค้นฐานข้อมูลเครื่องจุดขาย (Local DB เช่น `TPSTSalHD`, `TPSTSalDT`, `TPSTSalRC`) เพื่อหาเอกสารที่ `FTStaSentOnOff` มีค่าเป็น '0' (หมายถึงยังไม่ได้ซิงค์)
  4. วนลูปนำข้อมูลที่พบไปทำการ `INSERT` หรือ `UPDATE` เข้าสู่ฐานข้อมูลเซิร์ฟเวอร์
  5. หากอัปเดตสำเร็จ จะกลับมาอัปเดต `FTStaSentOnOff` ใน Local DB เป็น '1'
  6. สุดท้ายเรียกใช้ `W_UPDxUpdatePoint` เพื่อจัดการเรื่องคะแนนสะสมของสมาชิก

### 2. `W_UPDxUpdatePoint` (การประมวลผลคะแนนสมาชิก)
- **วิธีการ (Method/Type):** Internal Subroutine (อยู่ใน `wMain.frm`)
- **ข้อมูลที่ส่งเข้า (Request Payload):** ไม่มี
- **ข้อมูลส่งกลับ (Response Payload):** ไม่มี
- **สรุปตรรกะการทำงาน (Core Logic):**
  1. สืบค้นฐานข้อมูล Local DB ที่ตาราง `TPSTSalePoint` เพื่อหาคะแนนที่ยังไม่ได้ประมวลผล (`FTRemark = '0'`)
  2. จัดกลุ่มคะแนนตามรหัสสมาชิกและวันที่
  3. สำหรับแต่ละกลุ่ม ระบบจะเชื่อมต่อไปยังเซิร์ฟเวอร์เพื่อตรวจสอบคะแนนคงเหลือที่ตาราง `TCNMMallCard`
  4. หากมีข้อมูลสมาชิกอยู่แล้ว จะทำการบวกคะแนนเพิ่ม (`FCEarned`) แต่ถ้ายังไม่มี จะ Insert เรคคอร์ดสมาชิกใหม่
  5. บันทึกประวัติการเปลี่ยนแปลงคะแนนที่ตาราง `TPSTBPHis` บนเซิร์ฟเวอร์
  6. อัปเดต `FTRemark` ของตาราง `TPSTSalePoint` ใน Local DB เป็น '1' เพื่อระบุว่าดำเนินการแล้ว

### 3. `SP_PRCbToken` (การสร้าง Token แทนข้อมูลบัตรเครดิต/บัตรประชาชน)
- **วิธีการ (Method/Type):** Public Function (อยู่ใน `mMain.bas`)
- **ข้อมูลที่ส่งเข้า (Request Payload):** `ptCardNo` (String) - ข้อมูลบัตรต้นฉบับ
- **ข้อมูลส่งกลับ (Response Payload):** `ptRefCardNoToken` (String - ส่งกลับแบบ ByRef)
- **สรุปตรรกะการทำงาน (Core Logic):**
  1. รับข้อมูลที่มีความละเอียดอ่อน (เช่น เลขบัตร)
  2. ตรวจสอบในตาราง `TPSTTokenLst` ของเซิร์ฟเวอร์ว่ามี Token ของหมายเลขนี้อยู่แล้วหรือไม่
  3. หากมีอยู่แล้ว จะดึง Token เก่ากลับไปใช้งาน
  4. หากยังไม่มี ระบบจะสร้าง Token ใหม่ และ Insert จับคู่ค่าต้นฉบับ (`FTValue`) เข้ากับ `FTToken` ใน `TPSTTokenLst`
  5. ส่งคืนรหัส Token กลับไปยังโปรแกรมหลักเพื่อใช้บันทึกในตารางธุรกรรม (Transaction) แทนเลขบัตรจริง

### 4. `W_DATbInsertData` และ `W_DATbUpdateData` (ฟังก์ชันซิงค์ DB ส่วนกลาง)
- **วิธีการ (Method/Type):** Internal Functions (อยู่ใน `wMain.frm`)
- **ข้อมูลที่ส่งเข้า (Request Payload):** `porsTemp` (Recordset ชุดข้อมูล), `ptTblName` (String ชื่อตาราง)
- **ข้อมูลส่งกลับ (Response Payload):** Boolean (สถานะสำเร็จ/ล้มเหลว)
- **สรุปตรรกะการทำงาน (Core Logic):**
  1. เป็นฟังก์ชันเครื่องมือ (Helper) สำหรับสร้างสคริปต์ `INSERT INTO` หรือ `UPDATE` อัตโนมัติจากชุด Recordset ที่อ่านขึ้นมา
  2. ลูปหลักใช้ฟังก์ชันนี้บ่อยครั้งเพื่อโคลนข้อมูลจาก Local POS ไปยัง Server โดยไม่ต้องเขียนชื่อฟิลด์ทุกฟิลด์แบบฮาร์ดโค้ด
